1. WHAT IS OTA?
================================================================================
OTA = Online Travel Agency (like MakeMyTrip, Goibibo, Booking.com).
Our OTA partner is BOOKINGJINI. They act as a middleman - customers book 
tours/hotels through Bookingjini's platform, and Bookingjini sends that 
booking data to our system via a "webhook" (an API call).


2. WHAT WAS THE PROBLEM? (From Manoj's Email)
================================================================================
Bookingjini said:
  - Their side is DONE (endpoint setup, authentication, schema alignment)
  - Booking confirmations ARE being generated on their end
  - BUT the data is NOT saving into OUR database

ROOT CAUSE: 
  Our backend had NO webhook endpoint to RECEIVE booking data from Bookingjini.
  We only had code to FETCH hotel listings from them (one-way, read-only).
  There was no code to accept and save incoming bookings.

4. OUR WEBHOOK ENDPOINTS (LIVE ON RENDER)

  ENDPOINT                              METHOD    AUTH NEEDED?    PURPOSE
  ─────────────────────────────────────────────────────────────────────────
  /api/ota/webhook/health               GET       No             Health check / connectivity test
  /api/ota/webhook/booking              POST      Yes            Receive booking confirmations
  /api/ota/webhook/cancellation         POST      Yes            Receive cancellation notifications

  Authentication: Bookingjini must send a header:
